# 🧠 TARS AI Fusion: Multi-Model AI Interface Setup Guide

This project aims to unify multiple artificial intelligence models (such as DeepSeek, Gemini, Qwen, Claude, etc.) under a single ecosystem using **OpenRouter**, **9router**, and **Open WebUI**. By utilizing the "Fusion" module, it integrates these models to achieve maximum performance and virtually eliminate hallucinations.

## 🚀 Features
* **Single API for Multiple Models:** Manage dozens of LLMs with a single API key via OpenRouter.
* **Fusion Technology:** Zero-hallucination outputs by having a "Judge" model verify the data generated by "Worker" models via 9router.
* **Modern UI:** A fast, mobile-friendly, and ChatGPT-like user experience powered by Open WebUI.
* **Flexible Access:** 24/7 accessibility through a Localhost network or a remote VDS.

## 📋 Prerequisites
Before starting the installation, ensure that the following are installed on your system:
* [Python 3.11](https://www.python.org/downloads/release/python-3110/) (Required for Open WebUI)
* [Node.js](https://nodejs.org/en/download) (Required for 9router)
* **Operating System:** Windows or Zorin OS / Debian-based Linux distributions.
* *(Optional)* A VDS (Virtual Dedicated Server) is recommended if you want global access to the system at all times.

## 🛠️ Installation Steps

### Step 1: OpenRouter Configuration
1.  Go to [OpenRouter](https://openrouter.ai/) and create an account.
2.  Navigate to your Profile, select **API**, and generate a new API key.
3.  *(Optional)* If you plan to use premium models, add balance via the **Add Credit** section. You can skip this step if you are strictly using free models.

### Step 2: Open WebUI Installation
We will install Open WebUI as our primary front-end interface.

**For Windows Users:**
Open Command Prompt (CMD) as Administrator and run the following command:
`python -m pip install open-webui`

*Note: The download may take 5-10 minutes depending on your connection. If the command fails, navigate to `cd C:\Users\YOUR_USERNAME\AppData\Local\Programs\Python\Python311\Scripts` and try again.*

**For Linux Users:**
Open your terminal and run:
`pip3 install open-webui`

Once the installation is complete, start the server with:
`open-webui serve`

When the loading bars finish, open your browser and go to `http://localhost:8080` to access the Open WebUI panel and create an admin account.

### Step 3: 9router Installation (For Fusion)
To blend multiple models, we need to install the [9router](https://9router.com/) system.
Open your CMD or Terminal with administrator privileges and run these commands sequentially:
`npm install -g 9router`  
`9router`

1.  Select the **Web UI** option from the terminal menu.
2.  Log in on the browser page using the default password `123456` (Make sure to change this later).
3.  Go to the **Providers** tab, select **OpenRouter**, paste the API key you generated in Step 1, and click save.

### Step 4: Selecting Models and Fusion Configuration
1.  Navigate to the **Models** section in the 9router interface.
2.  Search for the models you want to use via OpenRouter (e.g., `deepseek/deepseek-v4-pro`) and copy the exact ID name.
3.  Go to **Providers > OpenRouter > Add Model** in 9router, paste the ID, and add it. Test the model to ensure it responds correctly.
4.  Review current model benchmarks on *[Artificial Analysis](https://artificialanalysis.ai/models)* to select 3 or 4 robust "Worker" models.

### Step 5: Creating TARS Fusion Combos
Go to the **Combos** tab on 9router and click **Create Combo** to blend your models. Don't forget to activate the `Fusion` mode for the combo.

⚠️ **Important Note (Cost Warning):** Since your "Judge" model will read and summarize massive amounts of data coming from the worker models, the "Input" cost can be high. To keep costs low, select a relatively cheap but intelligent model (e.g., DeepSeek V4 Pro) as your Judge.

### Step 6: Connecting the Systems and Exposing to Network
1.  Go to the **Endpoint** section in the 9router UI and copy the **API Key** and **URL**.
2.  Return to your Open WebUI panel (`localhost:8080`).
3.  Navigate to **Profile > Admin Panel > Settings > Connections**.
4.  Add a new connection under the OpenAI API section, paste the URL and API Key from 9router, and hit save.

**Network Access (Optional):**
To access the interface from other devices (Tablets/Phones) on the same network, type `ipconfig` (or `ip a` for Zorin OS) in your terminal to find your local IPv4 address (e.g., `192.168.x.x`). You can log in from your phone by navigating to `192.168.x.x:8080`. For remote access outside your home, consider using DuckDNS or Cloudflare Tunnels.

### ⚙️ Automation (One-Click Start)
You can easily start both Open WebUI and 9router simultaneously using the script files included in this repository.

*   **Windows Users:** Simply double-click the `start.bat` file.
*   **Linux Users:** Run `./start.sh` in your terminal (make sure to grant execution permissions first with `chmod +x start.sh`).
